    Item 12: Always override toString

    Programmer 時常會將物件的訊息印出來，以幫助診斷程式的行為。實作輸出有意義有助於 Programmer 閱讀的訊息有助於診斷問題。

    在自動化測試，錯誤訊息像

    Assertion failure: expected {abc, 123}, but was {abc, 123}.

    對除錯的幫助有限。

    其他補充

    • 輸出物件的所有重要資訊
    • 訊息量太多的 field，提供 summary 即可
